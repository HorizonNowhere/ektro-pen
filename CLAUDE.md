## 一、产品定义

**EKTRO** 是一个拼音输入法。它的**反叛性主张**：

> 输入法不该让你选字。候选窗是机器不够聪明的赔礼道歉。
> 30 年来，没人把"每打 4 个字就要看一次候选窗"当成 bug。
> 我们把它当 bug。

### 核心交互

```
   普通打字：
   ─────────
     输入 → inline 直接显示中文，无候选窗，无中断

   纠错：
   ─────
     错了 → 按 Tab 切换替代候选 → 自动学习

   长句预测：
   ─────────
     停顿 >300ms → 浮现淡灰续写 → 按 Tab 接受

   后台学习：
   ─────────
     无显式 query。最近 24h 输入即上下文。

   应急通道：
   ─────────
     长按 Ctrl → 唤起传统候选窗（99% 用不到）
```

## 二、范围铁律

### ✓ v1 做这些

- 拼音输入（全拼优先；双拼留 v0.2）
- 端侧个性化 rerank（~20M GRU）
- 上下文锁定（最近 24h 输入）
- Tab 切换替代候选
- 淡灰下一句预测（Qwen3-0.6B IQ4_XS 端侧）
- SQLite 数据明文，可视/可导/可删
- 双进程崩溃隔离（IME 前端 ⇄ Core 后端）

### ✗ v1 不做（无例外，PR 直接拒）

- 候选窗默认显示
- 桌宠 / Live2D
- Chat 浮窗
- 语音输入
- 云端 LLM 调用
- 表情 / 颜文字 / 符号面板
- 皮肤商店
- 多设备同步
- 设置面板的 60 个开关（限制 ≤6 个）

> **想加新功能？先回答：它与"三戒"冲突吗？**
> 任何"yes"都意味着不做。

## 三、技术决策（无投票，已定）

| # | 决策 | 理由 |
|---|------|------|
| 1 | **Fork weasel + librime** | 站在十年成熟引擎肩上，不重造 TSF 轮子 |
| 2 | **词库用万象拼音** | amzxyz 已用 32GB 语料训出 8-gram |
| 3 | **Rerank 用 GRU ~20M** | 实时通路必须 ≤30ms，LLM 太重 |
| 4 | **预测用 Qwen3-0.6B IQ4_XS** | llama.cpp 端侧，停顿 300ms 触发 |
| 5 | **候选窗代码保留但默认隐藏** | 长按 Ctrl 应急通道 |
| 6 | **双进程模型** | Core 崩溃 → IME 自动降级到纯 librime |
| 7 | **明文 SQLite** | 用户能看见和拥有自己的数据 |
| 8 | **Apache 2.0 许可** | 开源就绪友好 |

每条决策的展开理由见 `openspec/changes/ektro-mvp/design.md`（待建）。

## 四、性能 SLO（验收线）

任何破坏以下指标的 PR 都拒。

```
   首屏候选 (inline 字)        ≤ 50ms   P99
   GRU rerank 响应             ≤ 30ms   P99
   Qwen3 首 token 延迟         ≤ 200ms  P95
   内存常驻 (含 0.6B 量化)     ≤ 800MB
   CPU 空闲占用                < 1%
   每次 commit 落库异步        不阻塞输入
   网络请求数 (默认配置)        = 0
```

```
         ▸ 打 zip 自用
         ▸ 决定 Cycle 2 议题
```

> ✓ 没有崩溃 / 卡死 / 数据丢失。

## 七、与 AI 协作规范

### 模式与边界

| 模式 | 触发 | 允许 | 不允许 |
|------|------|------|--------|
| **探索** | `/openspec-explore` | 思考、调研、写决策文档 | 写代码 |
| **提案** | `/openspec-proposal` | 写 proposal/design/tasks | 跳过审核就执行 |
| **执行** | `/openspec-apply` | 按 tasks.md 执行 | 偏离已批准范围 |
| **归档** | `/openspec-archive` | 完成后归档变更 | 删除历史 |

### 通用规则

1. **中文交流**（默认）
2. **用 MCP 检索最新资料**，不依赖训练数据中可能过时的信息
3. **打字延迟永远是 P0**：任何让 P99 候选 >50ms 的改动 → 必须 design.md 评估
4. **三戒优先于功能**：如果一个 issue 想加的东西违反三戒，关闭 issue
5. **数据安全先于便利**：用户数据格式必须明文 + 可导出
6. **小步前进**：单个 PR ≤ 500 行变更（除非纯依赖更新）
7. **每个 Cycle 结尾派 Agent Swarm 验收一次**（≥4 个独立维度：代码质量 / 静默失败 / 测试 / 接口设计）。不通过则不进入下一 Cycle。来源：D-009。
8. **绝不静默吞错**：任何 `except` 必须接 `logger`。没 log 的 except 视为 bug。来源：D-009。
